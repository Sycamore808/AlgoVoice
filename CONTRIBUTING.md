# 贡献指南 / Contributing Guide

感谢您对 AlgoVoice 项目的兴趣！

---

## 📋 项目状态

本项目目前处于 **受限访问** 状态。代码可供查看和学习，但受到严格的使用限制。

---

## 🤝 如何参与

### 1️⃣ 报告问题 (Bug Reports)

如果您在使用过程中发现问题，欢迎提交 Issue：

1. 访问 [Issues 页面](https://github.com/Sycamore808/AlgoVoice/issues)
2. 点击 "New Issue"
3. 详细描述问题：
   - 问题现象
   - 复现步骤
   - 期望行为
   - 系统环境（操作系统、Python版本等）

### 2️⃣ 功能建议 (Feature Requests)

有好的想法？我们很乐意听取：

1. 在 [Issues](https://github.com/Sycamore808/AlgoVoice/issues) 中创建新issue
2. 标题使用 `[Feature Request]` 前缀
3. 描述：
   - 功能用途
   - 实现思路
   - 预期收益

### 3️⃣ 技术交流 (Discussions)

对项目设计或技术实现有疑问？

1. 访问 [Discussions](https://github.com/Sycamore808/AlgoVoice/discussions)
2. 选择合适的分类
3. 发起讨论

## 🔐 申请代码访问权限

如需获得完整代码访问权限或参与开发，请：

### 申请步骤

1. **创建申请Issue**
   - 访问 [Issues](https://github.com/Sycamore808/AlgoVoice/issues/new)
   - 标题：`[Access Request] 申请访问权限 - 您的姓名/组织`

2. **提供以下信息**
   ```
   - 姓名/昵称：
   - 所在机构/学校：
   - 技术背景：
   - 使用目的：
   - 计划贡献：
   - 联系方式：
   ```

3. **等待审核**
   - 我们会在 **3个工作日** 内回复
   - 审核通过后，将邀请您成为协作者

### 审核标准

我们优先考虑以下申请者：
- ✅ 有明确的研究或开发目的
- ✅ 具备相关技术背景
- ✅ 愿意贡献代码或文档
- ✅ 认同项目的开发理念

## 🌟 成为贡献者

一旦获得访问权限，您可以：

1. **Fork 项目**（如果允许）
2. **创建功能分支** (`git checkout -b feature/AmazingFeature`)
3. **提交更改** (`git commit -m 'Add some AmazingFeature'`)
4. **推送分支** (`git push origin feature/AmazingFeature`)
5. **创建 Pull Request**

### 代码规范

- 遵循 PEP 8 Python 代码规范
- 添加适当的注释和文档
- 编写测试用例
- 确保所有测试通过
